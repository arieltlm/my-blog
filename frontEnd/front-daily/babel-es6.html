# 2). babel基本用法

# 5).babel-polyfill

Babel默认只转换新的JavaScript句法（syntax），而不转换新的API，比如Iterator、Generator、Set、Maps、Proxy、Reflect、Symbol、Promise等全局对象，以及一些定义在全局对象上的方法（比如Object.assign）都不会转码。

举例来说，ES6在Array对象上新增了Array.from方法。Babel就不会转码这个方法。如果想让这个方法运行，必须使用babel-polyfill，为当前环境提供一个垫片。

知识来源 [Babel]代码转码器的使用说明 (opens new window) ES6环境搭建(Node & babel) (opens new window) 说说 Babel 的 babel-node 工具 (opens new window) 安装了babel-cli之后不能够使用babel-node？ (opens new window)

# 3.升级babel7

新版本的Babel7中对官方提供的依赖使用了命名空间@babel； 所以之后的都需要安装为@babel/cli,@babel/preset-env，@babel/preset-react等等，@babel/node需要自行安装了。

目前出来的转化插件都需要Babel7才可用

具体可以查看babel文档（官方） (opens new window)；

转化插件可查看babel插件列表（官方） (opens new window)

实际使用详细查看官网Usage Guide (opens new window):

  • 最基础安装下面几个包：
npm install --save-dev @babel/core @babel/cli @babel/preset-env
npm install --save @babel/polyfill
1
2
  • 将src下面的文件都转码输出到lib下
./node_modules/.bin/babel src --out-dir lib
1

还可以增加插件

./node_modules/.bin/babel src --out-dir lib --plugins=@babel/plugin-transform-arrow-functions
1

原文：Just like with plugins, you can create your own presets too to share any combination of plugins you need. For our use case here, there's an excellent preset named env

就如同插件那样，可以配置很多个preset组合起来用，此次就有一个很棒的包@babel/preset-env,不再需要像之前那样，增加es7各个阶段的、babel-preset-react、babel-preset-2015，只需要一个preset-env即可

./node_modules/.bin/babel src --out-dir lib --presets=@babel/env
1

原文：Without any configuration, this preset will include all plugins to support modern JavaScript (ES2015, ES2016, etc.). But presets can take options too. Rather than passing both cli and preset options from the terminal, let's look at another way of passing options: configuration files.

  • 配置文件babel.config.json

Configure Babel (opens new window)

{
  "presets": [
    [
      "@babel/env",
      {
        "targets": {
          "edge": "17",
          "firefox": "60",
          "chrome": "67",
          "safari": "11.1",
        },
        "useBuiltIns": "usage",
      }
    ]
  ]
}
